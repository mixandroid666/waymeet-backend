Recommendation: Start with Plan B (Railway/Render + Neon or Supabase + Upstash + Cloudflare R2). It stays near free-tier cost, needs no code changes — only env-var swaps for DATABASE_URL, REDIS_URL, and S3_* — gives HTTPS API access from anywhere, and handles the always-on worker cleanly. Drop to Plan A to squeeze cost to ~$5; move to Plan C when traffic demands autoscaling.

Before you migrate

  • Your code uses the AWS S3 SDK, so any S3-compatible target (R2, Supabase, MinIO) works by setting S3_ENDPOINT + keys — no rewrite.
  • Confirm the Postgres host allows CREATE EXTENSION postgis; — Neon, Supabase, and Cloud SQL all do; many bargain hosts do not.
  • Build the worker image with --build-arg CMD=worker; the api uses the default CMD=api.
  • Keep Firebase FCM and Resend as external SaaS — only their credentials/keys move to the host's secret store.
