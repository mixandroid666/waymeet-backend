# waymeet-backend

Go backend for **waymeet**, the travel social app (frontend: `../waymeet-flutter`).

Production-grade, scalable, built as a **modular monolith** with a separate
background **worker** â€” clean module seams so any hot component can later be
extracted into its own service without untangling.

## Stack

| Concern        | Choice                                              |
| -------------- | --------------------------------------------------- |
| Language       | Go 1.26                                             |
| HTTP           | stdlib `net/http` (ServeMux, method patterns) â†’ chi when needed |
| Database       | PostgreSQL 16 + **PostGIS** (geospatial discovery)  |
| DB access      | **sqlc** + **pgx** (type-safe SQL)                  |
| Migrations     | golang-migrate                                      |
| Realtime chat  | WebSockets + **Redis pub/sub** (scales across instances) |
| Cache / queue  | Redis (+ asynq for jobs)                            |
| Auth           | JWT (access/refresh) + OAuth2 (Google/Facebook/TikTok) |
| Media          | S3 / MinIO + presigned uploads; ffmpeg in worker    |
| Push           | FCM                                                 |
| Observability  | slog, OpenTelemetry, Prometheus                     |

## Layout

```
cmd/
  api/                 HTTP API entrypoint (runnable now)
  worker/              background jobs: media, notifications, feed precompute
internal/
  platform/            cross-cutting infra
    config/            env config
    logging/           slog setup
    httpx/             JSON + error envelope + middleware
    server/            router + route registration
    storage/dbgen/     (generated by sqlc â€” not committed until generated)
  auth/  user/  feed/  location/  chat/  media/  notification/
                       feature modules (currently doc.go seams)
db/
  migrations/          golang-migrate SQL (0001_init)
  queries/             sqlc source queries (users, location, feed)
api/openapi.yaml       client-facing API contract
```

Each `internal/<module>` maps to a screen/feature in the Flutter app â€” see each
package's `doc.go`.

## Quickstart

```bash
cp .env.example .env

# Build & run (no external deps yet â€” stdlib only):
go build ./...
make run                 # API on :8080
curl localhost:8080/healthz   # {"status":"ok"}

# Bring up infra when you start wiring the database/chat/media:
make up                  # postgres+postgis, redis, minio
make migrate-up          # needs golang-migrate + DATABASE_URL
```
